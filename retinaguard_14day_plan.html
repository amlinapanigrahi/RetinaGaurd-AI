Phase 1 — Setup & Data Pipeline Days 1–3 ▼
Day 1
Environment & repo setup
Create GitHub repo with clean folder structure (data/, models/, api/, notebooks/). Set up a virtual environment with Python 3.10. Install PyTorch, torchvision, scikit-learn, XGBoost, FastAPI, Uvicorn, Pillow, pandas, and albumentations. Write a requirements.txt immediately.
PyTorchFastAPIXGBoost
Day 2
Download & explore datasets
Download APTOS 2019 Blindness Detection from Kaggle (fundus images, 5 severity grades). Download Pima Indians Diabetes Database (tabular). Run EDA in a Jupyter notebook — check class distribution, image quality, missing values. APTOS has heavy class imbalance (most images are grade 0) — note this, you'll handle it on Day 3.
APTOS 2019Pima Diabetes
Day 3
Data preprocessing & augmentation
Resize all fundus images to 224×224. Write a PyTorch Dataset class with augmentation (random flip, rotation, color jitter). Handle class imbalance using WeightedRandomSampler or class weights — this is critical for good results. For tabular: handle missing values, normalize features with StandardScaler. Save processed tabular data as a clean CSV.
Class imbalance fixalbumentations
✓ Milestone: Both datasets are clean, EDA notebook committed, DataLoader runs without errors
Phase 2 — Deep Learning Model (ResNet-50) Days 4–6 ▼
Day 4
Build & train baseline model
Load pretrained ResNet-50 from torchvision.models. Replace the final FC layer with a new linear layer outputting 5 classes (DR grades 0–4). Freeze all layers except the last 2 blocks + final FC. Train for 5 epochs on Kaggle GPU (use Kaggle notebooks — free T4). Log train/val accuracy and loss. Don't aim for perfect results yet — just get the pipeline running end-to-end.
Transfer learningKaggle GPU
Day 5
Fine-tune & improve
Unfreeze all layers, use a lower learning rate (1e-4) with AdamW optimizer. Add a learning rate scheduler (CosineAnnealingLR). Train for 10 more epochs. Plot confusion matrix and per-class accuracy — expect worse performance on rare grades (1, 2). That's okay and realistic. Compute quadratic weighted kappa score (that's the official APTOS metric — use sklearn.metrics.cohen_kappa_score).
Kappa scoreAdamW + scheduler
Day 6
Save model + inference script
Save the best checkpoint as resnet50_dr.pth using torch.save(). Write a standalone predict_image(image_path) function that loads the model, preprocesses the image, and returns the severity grade + confidence score. Test it on 10 images manually. Write a brief model card (accuracy, kappa score, dataset used).
Model cardInference script
✓ Milestone: ResNet-50 runs inference correctly on a single image, model card written, checkpoint saved
Phase 3 — XGBoost Model + FastAPI Skeleton Days 7–9 ▼
Day 7
Train XGBoost risk classifier
Use Pima dataset features: pregnancies, glucose, blood pressure, skin thickness, insulin, BMI, diabetes pedigree, age. Train XGBoost with cross-validation. Tune max_depth, n_estimators, and learning_rate using GridSearchCV (keep it small — 2–3 values each). Output: risk score (0–1 probability) and binary risk flag. Save model with joblib.dump(). Target AUC-ROC above 0.80.
XGBoostGridSearchCVAUC-ROC
Day 8
FastAPI skeleton + Pydantic schemas
Create main.py with FastAPI app. Define a PatientData Pydantic model (age, HbA1c, blood_pressure, years_with_diabetes, glucose, bmi). Create two placeholder endpoints: POST /predict/tabular (accepts JSON, returns risk score) and POST /predict/image (accepts file upload, returns DR grade). Wire up the XGBoost model to the tabular endpoint. Test with curl or the auto-generated Swagger UI at /docs.
PydanticFastAPI /docs
Day 9
Wire image endpoint + model loading
Wire ResNet-50 to the /predict/image endpoint. Load both models once at startup using FastAPI's lifespan context (not on every request — that's a common beginner mistake). The image endpoint should accept multipart/form-data, preprocess the image in memory, and return grade + confidence. Add basic error handling (wrong file type, missing fields).

Phase 4 — Integration, Mismatch Logic & Testing Days 10–12 ▼
Day 10
Unified /analyze endpoint
Create a single POST /analyze endpoint that accepts both the patient JSON and fundus image together. It should call both models internally and return a combined response: DR grade, confidence, systemic risk score, risk flag, and a mismatch_alert boolean. The mismatch logic: if image grade >= 3 (Severe/Proliferative) AND tabular risk_flag == 0 (low risk), set mismatch_alert = True with a warning message.
Mismatch alert logicUnified endpoint
Day 11
Write tests + Dockerize
Write at least 5 pytest tests: test the tabular endpoint with valid/invalid inputs, test the image endpoint with a sample fundus image, test mismatch alert trigger. Then write a Dockerfile (python:3.10-slim base, COPY requirements.txt, RUN pip install, COPY app, CMD uvicorn). Build and run it locally — if it works in Docker, it's portable. This is a huge plus for internship panels.
